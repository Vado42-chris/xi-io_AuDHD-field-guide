# Batch 3 Execution Workbook

## Why this document exists

The repo now has:
- a Batch 1 execution workbook and closeout template
- a Batch 2 execution workbook and closeout template
- a Batch 1 to Batch 2 comparison template
- a pilot readiness progression ladder
- a broader readiness planning checkpoint template

That is enough to run, close, compare, and assess the first two pilot rounds.

The next practical need is a **dedicated workbook for Batch 3**.

Batch 3 should not just repeat Batch 2.
It should test whether the product is:
- stabilizing in the right ways
- holding up after earlier adjustments and carry-forward decisions
- showing enough consistency to justify opening broader readiness planning

This workbook is the working sheet for Batch 3.

---

# 1. Batch 3 intent

## Batch 3 primary goal

Confirm whether:
- the product is stabilizing rather than merely accumulating pilot activity
- the core support loop, Help Now, Learn Me, personalization, and trust posture are holding up with enough consistency to justify broader readiness planning discussion

## Batch 3 secondary goal

Watch for:
- whether earlier improvements remain durable
- whether repeated-use value is becoming more consistent across participants
- whether unresolved issues are narrowing or merely shifting
- whether the pilot is now producing more confirmation than surprise

## Suggested Batch 3 size

- 3 to 5 trusted participants

# 5. What Batch 3 is specifically testing

## Stabilization value

Questions:
- Are the main product surfaces becoming more consistently understandable?
- Are earlier improvements still visible, or are they fading under new complexity?
- Does the pilot now generate more confirmation than surprise?

Notes:  

## Broader readiness signal

Questions:
- Is the product starting to show enough stability for broader readiness planning to be discussed?
- Are major blockers no longer dominating the signal?
- Are remaining issues narrowing in scope?

Notes:  

## Carry-forward checks from Batch 1 and Batch 2

Questions:
- Did known friction points continue improving?
- Did any accepted risks become real blockers?
- Did any “improvements” from earlier batches prove fragile?

# 7. Session checklist, Batch 3 emphasis

Use this during each Batch 3 session.

## Before session

- [ ] Participant is appropriate for Batch 3 scope
- [ ] Any relevant earlier caveat changes are reflected in briefing
- [ ] Observation template ready
- [ ] Facilitator is clear on Batch 3 focus areas

## During session

- [ ] Observe whether the user still understands the core loop without heavy support
- [ ] Observe whether Help Now still feels calm and readable
- [ ] Observe whether Learn Me has durable value
- [ ] Observe whether personalization feels consistently meaningful
- [ ] Observe whether trust language still lands correctly
- [ ] Capture any sign that the product is stabilizing or not stabilizing
- [ ] Capture any new blocker or repeated friction

## After session

- [ ] Observation template completed
- [ ] Follow-up sent
- [ ] Tracker updated
- [ ] Notes marked ready for synthesis
- [ ] Any blocker flagged immediately

# 11. Honest reminder

Batch 3 exists to test whether the product is becoming stable enough to justify harder readiness questions.

If Batch 1 asks, “Can people get it?”
If Batch 2 asks, “Does it keep making sense?”
Batch 3 asks, “Is it stabilizing enough that broader planning is now responsible to consider?”

That is the standard for this workbook.
