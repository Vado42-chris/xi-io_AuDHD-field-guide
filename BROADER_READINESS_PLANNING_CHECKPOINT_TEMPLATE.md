# Broader Readiness Planning Checkpoint Template

## Why this document exists

The repo now has:
- a pilot readiness checkpoint
- a progression ladder
- Batch 1 and Batch 2 execution workbooks
- Batch 1 and Batch 2 closeout templates
- a Batch 1 to Batch 2 comparison template

That is enough to run, close, compare, and assess early trusted pilot work.

The next practical need is a **broader readiness planning checkpoint**.

This document is not for routine early pilot use.
It should only be used when the pilot evidence may be strong enough to justify asking:
- should broader readiness planning begin?
- what evidence is already strong enough?
- what still blocks broader readiness planning?
- what additional proof would still be required before any wider exposure is responsible?

This checkpoint exists to prevent premature expansion language.

# 7. What broader readiness planning would and would not mean

## It would mean

Examples:
- planning a wider evaluation phase
- refining participant or tester scope
- defining stronger readiness criteria for broader use
- scoping what infrastructure, QA, and messaging would be needed next

Notes:  

## It would not mean

Examples:
- claiming the product is finished
- claiming the product is validated in a broad or clinical sense
- skipping unresolved blockers
- treating pilot success as final proof

# 13. Honest reminder

Broader readiness planning is not a reward for running a good pilot.
It is a decision that the evidence has become strong enough to justify asking harder questions about broader exposure.

If that evidence is not there, this checkpoint should say so plainly.
