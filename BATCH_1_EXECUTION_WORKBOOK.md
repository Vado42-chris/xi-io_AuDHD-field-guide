# Batch 1 Execution Workbook

## Why this document exists

The repo now has:
- a pilot operations index
- a pre-pilot regression gate
- participant and facilitator materials
- observation templates
- synthesis and triage templates
- rollout and batch planning templates

That is enough structure to run a pilot.

The next practical need is a **single workbook for Batch 1 itself**.

This document is meant to be used as the working sheet for the first pilot batch.
It turns the broader pilot stack into an executable sequence with explicit completion points.

---

# 1. Batch 1 intent

## Batch 1 primary goal

Confirm whether:
- the core support loop is understandable without heavy coaching
- Help Now feels calmer and clearer in real use
- trust language feels appropriately cautious rather than too strong or too vague

## Batch 1 secondary goal

Watch for early signals around:
- whether Learn Me feels worth revisiting
- whether personalization is noticeable and meaningful
- whether any blocker appears quickly enough to pause Batch 2

## Suggested Batch 1 size

- 3 trusted participants

# 5. Suggested day-by-day sequence

## Day 0, Gate and setup

- [ ] Regression gate completed
- [ ] Accepted-risk items documented, if any
- [ ] Batch 1 goals confirmed
- [ ] Participant shortlist finalized
- [ ] Rollout tracker updated
- [ ] Batch schedule updated

Notes:  

## Day 1, Outreach

- [ ] Send invitation to Participant 1
- [ ] Send invitation to Participant 2
- [ ] Send invitation to Participant 3
- [ ] Track accept / decline status
- [ ] Move any reserve participants into place if needed

Notes:  

## Day 2, Scheduling and briefing

- [ ] Confirm accepted participants
- [ ] Send participant brief to accepted participants
- [ ] Schedule session windows
- [ ] Confirm facilitator availability
- [ ] Confirm session note documents are ready

Notes:  

## Day 3 to Day 5, Sessions

For each session:
- [ ] Run session using facilitator runbook
- [ ] Capture notes using observation session template
- [ ] Mark session complete in rollout tracker
- [ ] Send post-session follow-up
- [ ] Mark whether urgent blocker was observed

Notes:  

## Day 6, Synthesis

- [ ] Gather all completed session notes
- [ ] Fill findings synthesis template
- [ ] Identify recurring successes
- [ ] Identify recurring friction points
- [ ] Identify any pilot blockers

Notes:  

## Day 7, Triage and decisions

- [ ] Convert meaningful issues into triage records
- [ ] Create decision log entries for real product or pilot calls
- [ ] Decide whether Batch 2 should proceed, proceed with caveats, or pause
- [ ] Update batch schedule accordingly

Notes:

# 10. Honest reminder

Batch 1 is not a victory lap.
It is the first real proof point.

The goal is not to show that the product is already finished.
The goal is to learn clearly enough from the first real batch that the next move is justified by evidence instead of momentum.
