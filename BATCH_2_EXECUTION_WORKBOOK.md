# Batch 2 Execution Workbook

## Why this document exists

The repo now has:
- a Batch 1 execution workbook
- a Batch 1 closeout report template
- synthesis, triage, and decision templates
- rollout and batch planning templates

That is enough to run and close the first pilot batch.

The next practical need is a **dedicated workbook for Batch 2**.

Batch 2 should not just repeat Batch 1 blindly.
It should test what Batch 1 did not prove yet, and it should explicitly account for any:
- blockers
- accepted risks
- caveat changes
- product adjustments
- facilitator learnings

This workbook is the working sheet for Batch 2.

---

# 1. Batch 2 intent

## Batch 2 primary goal

Confirm whether:
- Learn Me feels worth revisiting after more than one use
- personalization feels more meaningful after early exposure
- trust language still lands correctly when the product is no longer brand new

## Batch 2 secondary goal

Watch for:
- whether Batch 1 friction points improved
- whether new issues emerge after repeated use
- whether support view preference memory and pinned comfort behavior remain understandable over time
- whether the app still feels calm when used more than once

## Suggested Batch 2 size

- 3 to 5 trusted participants

# 5. What Batch 2 is specifically testing

## Repeated-use value

Questions:
- Does the product still feel useful after first exposure?
- Does Learn Me become more meaningful after multiple interactions?
- Does the user understand the product better over time, or does confusion compound?

Notes:  

## Personalization value

Questions:
- Do pinned comforts feel more useful after continued exposure?
- Do custom state labels matter more over time?
- Does support view preference memory feel helpful or invisible?

Notes:  

## Batch 1 carry-forward checks

Questions:
- Did Batch 1 friction points improve?
- Did any accepted risks prove harmless or become bigger issues?
- Did any wording changes land better or worse?

# 7. Session checklist, Batch 2 emphasis

Use this during each Batch 2 session.

## Before session

- [ ] Participant is appropriate for Batch 2 scope
- [ ] Any relevant Batch 1 caveat changes are reflected in briefing
- [ ] Observation template ready
- [ ] Facilitator is clear on Batch 2 focus areas

## During session

- [ ] Observe whether the user still understands the core loop
- [ ] Observe whether Learn Me feels more or less useful than before
- [ ] Observe repeated-use trust reactions
- [ ] Observe personalization value over time
- [ ] Capture any comparison to earlier impressions, if applicable
- [ ] Capture any new blocker or repeated friction

## After session

- [ ] Observation template completed
- [ ] Follow-up sent
- [ ] Tracker updated
- [ ] Notes marked ready for synthesis
- [ ] Any blocker flagged immediately

# 11. Honest reminder

Batch 2 exists to test whether the product still holds up after the first-contact glow is gone.

If Batch 1 asks, “Can people get it at all?”
Batch 2 asks, “Does it keep making sense, and does it keep feeling worth using?”

That is the standard for this workbook.
