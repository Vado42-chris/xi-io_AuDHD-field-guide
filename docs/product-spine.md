# Product Spine

## Product name
xi-io | AuDHD Field Guide

## Product promise
A state-aware, local-first interactive journal with ibal that helps users capture experiences, continue conversations over time, identify stressors and de-stressers, learn sensory preferences, and review patterns with increasing accuracy.

## Product boundaries
### This product is
- an interactive journaling and reflection tool
- a memory and retrieval system for user-authored experience
- a pattern-learning system with explicit confidence thresholds
- a support surface for low-demand, safe suggestions
- a local-first product with user-owned history and exports

### This product is not
- a score dashboard
- a generic chatbot shell
- a content feed or social platform

## Top-level product jobs
1. Help Now
2. Journal
3. Learn Me
4. Customize

## Trust rules
- custom suggestions do not unlock before the personalization threshold is met
- the user can correct tags, summaries, and inferred structure
- current state is always editable
- uncertainty is always a valid answer
- user data remains exportable and local-first by default

## Quality rules
- every feature must be understandable by a first-time user
- every primary flow must work without requiring free-text input
- every new feature must define what shared objects it reads and writes
- every slice must ship with empty, unsure, skip, and recovery paths
- accessibility is a baseline requirement

## Navigation lock
The new app shell must lock into these destinations only:
- Help Now
- Journal
- Learn Me
- Customize

Anything else must fit inside one of those categories.

## Success condition
The reset is successful when the product no longer feels like a telemetry-heavy concept dashboard and instead feels like a calm, usable, state-aware journaling product for repeated use.
