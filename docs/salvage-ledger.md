# Salvage Ledger

## Keep
### Brand kit
- color tokens
- typography direction
- calm dark visual language
- button feel
- panel surfaces

### Memory core
- threads
- messages
- journal entries
- export concept
- local-first persistence concept

### Assistant spine
- ibal identity
- continuation model
- node or mode concept, repositioned as assistive lenses
- source-grounded assistance where relevant

### Pattern engine concepts
- recency weighting
- trends
- contradictions
- load hints
- history analysis

## Refactor
- thread model to support current state and state transitions
- persistence into a dedicated storage layer
- pattern engine into quieter internal support logic
- naming into calmer primary UX copy
- export into a clearer user-facing vault and summary system
- assistant modes into task-specific flows inside Journal and Help Now
- theme tokens from inline HTML styles into reusable theme files

## Retire
- top-level navigation and page hierarchy
- assessment-first UX
- telemetry-heavy home
- score-centered product framing
- monolithic App.tsx as the application center
- dashboard-first interaction model

## Defer
- intel or news surfaces
- subscription and tier complexity
- advanced institutional or enterprise concepts
- non-core visual effects that add cognitive load

## Rule
No retained asset keeps its old product authority automatically. Retained assets must be re-validated against the new product spine and slice order.
