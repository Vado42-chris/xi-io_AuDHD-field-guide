# Slice Ledger

## Build order

### Slice 01
**Help Now + Current State**
- persistent global current state selector
- starter canonical states and customizable user labels
- state intensity
- Help Now launcher
- safe starter supports only
- support outcome logging
- post-send state confirmation
- thread inheritance from current state

### Slice 02
**Interactive Journal + Thread Continuity**
- thread list
- continue old conversations
- state-aware journal threads
- calmer ibal conversation UI
- save summary and tags
- editable thread metadata

### Slice 03
**Stressors, De-Stressers, and Sensory Capture**
- structured capture from threads and support flows
- user confirmation of extracted structure
- comfort wall foundation
- sensory support records
- helpfulness outcomes

### Slice 04
**Learn Me / Pattern Review**
- repeated patterns
- what helps most
- stressor and de-stresser review
- sensory profile review
- pattern confidence visibility

### Slice 05
**State Studio**
- edit default states
- add custom states
- map custom labels to canonical meanings
- choose favorites for quick selector

### Slice 06
**Threshold and Personalization Unlock**
- evidence rules
- confidence ladder
- unlock logic for tailored suggestions
- low-confidence vs high-confidence phrasing rules

## Slice rules
- no new slice begins until the current slice is end-to-end coherent
- every slice must define what shared objects it reads and writes
- every slice must define reusable components it depends on
- every slice must include empty, skip, unsure, and recovery paths
- every slice must pass accessibility review before complete

## Out of scope until core slices are stable
- growth features that increase complexity without improving repeated use
- parallel feature work across multiple top-level jobs
- cosmetic polish that avoids structural problems
- advanced feature branches with no dependency contract
