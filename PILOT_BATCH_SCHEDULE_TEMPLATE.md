# Pilot Batch Schedule Template

## Why this template exists

This template is meant to organize a small trusted pilot into explicit batches.

Its purpose is to keep the pilot from turning into one long blurry stream of sessions.
It helps define:
- which participants belong to which batch
- what each batch is meant to learn
- when sessions happen
- when synthesis happens
- when triage and decision logging should happen
- whether the pilot should continue to the next batch

Use one copy of this template per pilot round or per major pilot cycle.

# 4. Recommended cadence

A simple cadence for a small trusted pilot might be:

## Batch 1
Purpose:
- first-use comprehension
- Help Now clarity
- trust-language first reactions

## Batch 2
Purpose:
- Learn Me return-use value
- personalization value
- confusion that appears after repeated use

## Batch 3
Purpose:
- confirm whether earlier fixes or adjustments improved the right things
- check whether the product is becoming more stable or if new issues are emerging

This is only a suggested structure.
Adjust to the pilot size and goals.

---

# 5. Batch checkpoint questions

Ask these at the end of each batch:

- Did this batch answer its main question clearly enough?
- Did new pilot blockers appear?
- Did known blockers get better, worse, or stay unchanged?
- Is the next batch still justified without more product work first?
- Do participant-facing caveats need to change?

---

# 6. Pause conditions between batches

Pause before the next batch if:
- a blocking issue repeats across multiple sessions
- trust language is being misunderstood in a serious way
- the main support loop is breaking or being misunderstood too often
- the product is generating more confusion than signal
- the team cannot synthesize findings fast enough to keep the pilot honest

Notes:  

---

# 7. Honest reminder

A pilot batch is not just a scheduling unit.
It is a learning unit.

If a batch finishes and no real synthesis, triage, or decision step follows, the pilot is drifting.
This template exists to prevent that drift.
